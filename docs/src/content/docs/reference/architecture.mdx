## Data flow

```text
Bundled board-directory.yml + GitLab project members + issues
                                   |
                            background sync
                                   v
                    PostgreSQL snapshots
                              |
                   injected bootstrap JSON
                              v
                       React Board
```

Production traffic is ready only after directory, members, and board snapshots exist. Go injects the complete authenticated bootstrap into `index.html`; React pre-fills TanStack Query before the first render. `GET /api/v1/bootstrap` is the development fallback and 30-second background refresh.

## Dependency rule

```text
transport -> application -> domain
infrastructure -> application -> domain
composition root -> all concrete implementations
```

Domain knows no frameworks. Application use cases own narrow consumed ports and transaction boundaries. The file directory, GitLab, and PostgreSQL adapters translate external behavior. Architecture tests enforce this direction.

## Mutations

Card mutations update optimistic `issue_cache` and insert a `durable_operation` in one transaction. New cards receive a temporary negative IID. The worker sends current canonical title, Markdown description, `start_date`, `due_date`, labels, and the `assignee_ids` array to GitLab, replaces temporary IIDs after creation, and reconciles responses. Open lists use the scoped labels `Status::Waiting`, `Status::Inbox`, `Status::To Do`, `Status::Doing`, and `Status::Review`; Closed uses the issue state. Reads prioritize scoped status labels over legacy plain labels, which the next mutation removes. The join table preserves multiple assignees while deferred foreign keys allow temporary IIDs to become GitLab IIDs. Failed operations keep the optimistic state and can be retried.

## Frontend ownership

TanStack Query owns the bootstrap server snapshot. `features/onboarding` owns primary-team confirmation. `features/board` owns transient drafts, the detailed planning dialog, optimistic mutation, multiple-assignee reconciliation, drag/drop and keyboard controls. The Board uses the ordered `Wating`, `Inbox`, `To Do`, `Doing`, `Review`, and `Closed` lists. Team names, prefixes, labels, members, lists, and cards always come from bootstrap.

## Security

GitLab OAuth uses Authorization Code with PKCE and single-use encrypted state. Browser sessions are random opaque tokens in HttpOnly cookies; PostgreSQL stores keyed digests. Every valid use renews expiry to 14 days from that use. Authenticated mutations require both the session-bound CSRF token and an allowed Origin.
