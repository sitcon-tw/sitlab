The shared package owns reusable primitives and the SITCON browser token source. Feature workflows remain in `web/src/features`.

## Token contract

Product CSS consumes semantic `--sb-*` roles for surface, text, border, focus, action, assignment, and sync state. Shared primitives use `--pt-*` aliases backed by those same values. Raw color literals belong only in the token source, browser metadata, or a documented platform bridge. Both dark and light themes preserve the same hierarchy and semantic meaning.

The dark product header uses the official white SITCON logo shipped by the `sitcon-tw/2027` source. Its ink, paper, mist, acid, coral, and blue values inform the semantic roles; product code does not recreate or recolor the brand mark.

## Interaction contract

- Use `:focus-visible` for keyboard focus.
- Dialogs and drawers trap focus and return it to the trigger.
- Icon-only controls have accessible names and tooltips when the symbol is not universal.
- Production Board never replaces injected bootstrap with a loading page, skeleton, spinner, or empty columns.
- Healthy background sync, optimistic pending, processing, and success remain visually quiet. Offline snapshots and durable failures are visible and retryable.
- Card surfaces expose title, description preview, due date, and assignees. Team and status controls live in the detailed planning dialog with editable title, GFM Markdown edit/preview modes, multiple assignees, GitLab Start/Due dates, and the issue link. Markdown preview does not enable raw HTML.
- Controls keep stable dimensions so dynamic labels and counters do not shift the surrounding layout.

## Verification

Each public primitive has a Storybook story. Interactive components include keyboard and focus tests. Browser tests cover 320, 608, 928, and 1440 pixel viewports and reject incoherent overlap or long-text overflow.
