## Prerequisites

- Go 1.23.5
- Node.js 22.12 or newer and pnpm 11.11.0
- PostgreSQL 17
- `just` 1.40.0
- Docker with Compose for the production-shaped path

## GitLab setup

Create an OAuth application whose redirect URI exactly matches `SITCON_BOARD_GITLAB_OAUTH_REDIRECT_URL`. Provide a project access token able to read members, labels and issues in `sitcon-tw/2027`, plus the issue permissions needed by the operation worker.

The directory file must exist at `.sitcon/board-directory.yml`. The Docker build copies it into the image; changing it requires a new build and deployment, not a GitHub API token.

## Run from source

```bash
pnpm install
cp server/.env.example .env
# Set SITCON_BOARD_DATABASE_URL and GitLab credentials in .env.
just backend-migrate-up
```

Use separate terminals:

```bash
just backend-dev
```

```bash
just web-dev
```

The web server at `http://localhost:5173` proxies `/api` to the backend at `http://localhost:8080`.

For isolated browser design checks, run `VITE_SITCON_DEMO=true just web-dev`. This explicitly loads a test-only fixture and must not be used as a production data source.

## Production-shaped stack

```bash
cp deployments/docker/example.env deployments/docker/.env
# Replace every change-me and add GitLab credentials.
docker compose --env-file deployments/docker/.env -f deployments/docker/compose.yaml up --build
```

Open `http://localhost:3000`. Compose waits for PostgreSQL, completes migrations, then starts the application. Readiness remains false until the first directory/member/board sync succeeds.

## Validate

```bash
just ci
```

PostgreSQL integration tests use `SITCON_BOARD_TEST_DATABASE_URL` and remain a separate infrastructure-backed command: `just backend-test-integration`.
